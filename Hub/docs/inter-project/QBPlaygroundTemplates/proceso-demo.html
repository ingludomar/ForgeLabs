1

Panel de administración — QB Templates

Settings → Integrations → QB Templates

El panel centraliza todos los templates del sistema. Cada template está asociado a una operación QB y una sede. El badge Default indica el template que el Playground aplica automáticamente al seleccionar esa combinación. Desde aquí el administrador puede crear, editar, marcar como default o eliminar templates.

Panel QB Templates — 3 templates para ItemInventoryAdd · TEST. Badge Default indica el template activo por defecto.
2

Crear template — Paso 1: Selección de tipo, sede y nombre

Modal "Generar Template Base" — primera pantalla

Al hacer clic en + Generar template base, el administrador selecciona el tipo de operación QB (ej. Item — Add), la sede (TEST · RUS · REC · RBR · RMX) y asigna un nombre descriptivo al template. El dropdown de sede muestra únicamente las sedes registradas en el sistema.

Formulario inicial — dropdowns Tipo de Operación y Sede, campo Nombre del Template
Dropdown Tipo de Operación — Item Add/Mod/Query, Customer, Vendor, Invoice, PO, Bill, CC Charge y más
Dropdown Sede — TEST · RUS (Redsis USA) · REC (Redsis Ecuador) · RBR (Redsis Brasil) · RMX (Redsis México)
Paso 1 completado — Item — Add (ItemInventoryAdd) · TEST — Test Environment · nombre tpeNewItemAdd
3

Crear template — Paso 2: Selección de campos

Editor de checkboxes con todos los campos del contrato QB

El sistema carga automáticamente todos los campos disponibles en el contrato QB para esa operación. Los campos marcados con req están pre-chequeados y bloqueados — son obligatorios para QB Desktop y no pueden desmarcarse. Los campos opcionales pueden activarse o desactivarse libremente. El label de cualquier campo puede editarse haciendo clic sobre el texto.

Campos req pre-chequeados y bloqueados — Item Name, Sales Description, Sales Price, Purchase Description, Purchase Cost, Income Account, COGS Account, Asset Account
Campos opcionales — Quantity On Hand e Inventory Date seleccionados; Reorder Point, Manufacturer Part Number, Preferred Vendor, Class disponibles
4

QB Playground con template activo

Formulario reducido — solo los campos del template seleccionado

Al seleccionar una operación y sede que tiene template, el Playground muestra automáticamente el selector en el action bar. El formulario se adapta al template activo — en lugar de los 32 campos del contrato completo, el usuario ve únicamente los campos definidos en el template. El endpoint, versión y sede siguen visibles para referencia.

Selector tpeTestItemAdd activo — formulario reducido a 10 campos (vs 32 del contrato completo)
5

Formulario completo — opción "Todos los campos"

El usuario puede volver al contrato completo en cualquier momento

El selector incluye siempre la opción "Todos los campos". Al seleccionarla, el Playground muestra el contrato completo de QB Desktop con todos sus campos disponibles — incluyendo campos de referencia (Income Account, COGS Account, Asset Account, Preferred Vendor, Class, Parent Item, Unit of Measure, etc.). La preferencia guardada del usuario no se sobrescribe al usar esta opción.

Selector "Todos los campos" activo — formulario completo con datos de ejemplo: Item Name, Is Active, Sales Description, Sales Price, Purchase Cost, Quantity On Hand, Inventory Date, Reorder Point, Max Quantity…
Campos de referencia — COGS Account · Asset Account · Preferred Vendor · Class · Parent Item · Unit of Measure
Campos de referencia adicionales — Class · Parent Item · Unit of Measure · Sales Tax Code · Purchase Tax Code

Endpoints disponibles

API REST del módulo de templates — Redix backend (NestJS)
Método Ruta Descripción
GET /api/integration/qb-template Template activo + lista disponibles para un type+sede
GET /api/integration/qb-template/list Lista todos los templates activos (uso admin)
POST /api/integration/qb-template/generate Crear template con campos seleccionados
PUT /api/integration/qb-template/:publicId Actualizar nombre y/o campos de un template
PUT /api/integration/qb-template/:publicId/set-default Marcar como default para su type+sede
DELETE /api/integration/qb-template/:publicId Soft-delete — is_active = false
⟳

Flujo de resolución en el Playground

Secuencia de llamadas al seleccionar tipo + sede
GET /api/integration/template-preference

Obtiene el templateId guardado para el usuario actual (sede + operación). Respuesta en ~5ms desde DB local.

Promise.all — llamadas en paralelo

→ GET /api/integration/qb-template?type=X&sede=Y[&templateId=UUID] — campos del template desde DB de Redix
→ GET /api/integration/qb-contracts?type=X&sede=Y — requiredFields overlay desde LedgerOps

Resolución en el frontend

Si template ≠ null → formulario reducido con los campos del template.
Si template = null → formulario completo con todos los campos del contrato.

Selector visible

Aparece cuando existe al menos un template para la combinación activa. Incluye "Todos los campos" como opción siempre disponible.
